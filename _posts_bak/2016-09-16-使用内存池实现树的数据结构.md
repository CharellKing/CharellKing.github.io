## 树节点

通常我们定义一个树的节点，是这样的：

``` c++
struct TreeNode {
    T data;
    TreeNode* left;
    TreeNode* right;
};
```
left, right分别指向左孩子节点、右孩子节点，为内存地址。而在内存池中定义树节点，形式如下：

``` c++
struct TreeNode {
    T data;
    int left;
    int right;
};

```

由于内存池是以数组的形式存在，在数组中，存储的类型为TreeNode类型，TreeNode中的left、right，分别指向左孩子节点、右孩子节点，为数组的下标(范围为[0, n))。

## 内存管理

要实现的方法：

1. 申请内存池。

2. 销毁内存池。

3. 获取空闲节点。

4. 释放空闲节点。


``` c++
struct TreeNode {
	TreeNode(T data = T(), int left = -1, int right = -1):data(data), left(left), right(right) {
    }
	T data;
    int left, right;
};

class PoolMgr {
private:
    TreeNode* base;
    int capacity;
    int size;
    TreeNode* free_head;
public:
    PoolMgr():base(NULL), capacity(0), size(0), free_head(NULL) {
    }

    # 申请内存池
    void create_pool(size_t pool_capacity) {
        if (pool_capacity > 0) {
            base = new TreeNode(pool_capacity);
            capacity = pool_capacity;
        }
        return base;
    }

    # 销毁内存池
    void delete_pool() {
    	if (base) {
            delete [] base;
            base = NULL;
            capacity = 0;
            size = 0;
            free_head = NULL;
        }
    }

    # 使用空闲节点
    TreeNode* use_free_node() {
    	TreeNode* ret = NULL;
		if (size < capacity) {
			if (NULL == free_head) { // 连续的空闲节点
        		ret = base[size++];
    	    } else { // 零散的空闲节点
            	TreeNode* ret = free_head;
                free_head = (free_head->left >= 0 ? base[free_head->left]:NULL);
            }
        }
        return ret;
    }

    # 回收空闲节点
    void recycle_free_node(TreeNode* free_node) {
        if (NULL == free_head) {
        	free_head = free_node;
        	free_node->left = -1;
        } else {
        	free_node->left = (free_head - base);
            free_head = free_node;
        }
    }
};
```
这里内存池创建和销毁很简单，不用讲解；主要是使用空闲节点和回收空闲节点。我使用了一个回收栈(free_head)来管理零散的空闲内存节点。零散的空闲节点，主要是由于随机的使用和回收节点造成的。比如，我插入节点101、 102、 103、 104、 105，紧接着删除102,104,105，如下图：蓝色为使用的节点，红色为空闲节点：

![内存池的内存管理](/assets/picture/2016-09-16-内存池管理的利用回收.png)

其中空闲的节点，分为零散的空闲节点和连续的空闲节点。简单来说：零散的空闲节点是指曾经使用的节点，后来被回收了，大多情况下是非连续的；连续的空闲是指从来没用过的空闲节点，一定是连续的。在这里我使用一个栈结构（链表的形式）来为管理零散的空闲节点，其中next指针实质上为TreeNode的left，指向上一个空闲节点的下标。
